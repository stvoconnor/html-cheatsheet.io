HTML Structure Cheat Sheet

The main HTML structure

Tag Name of Element Description
<!DOCTYPE html> Doctype The Doctype decleration is required when creating a html document as it is used to make sure that the document follows all neccessary HTML specifications. Without the Doctype the document may render content incorrectly.
<head> The Head element Content that is not rendered in the browser but is imporant for website behaviour goes inside the head element. The exception being the <title> element, which places a title in the tab of a web page.
<body> The Body element All content that you can visually see when a website loads is placed within the body element.
<header> The Header element This is where all introductory content is placed, usually in the form of a banner, logo or company name located at the top of your page.
<nav> The navigation element This is where your websites navigation bar will be located. Often placed inside the header element but can be considered a seperate entity from your introductory content.
<main> The Main element All the unique and most relevant content that makes up the specific topic of your website ends up in this element.
<footer> The Footer element The footer is typically at the bottom of your page and usually displays copyright notices, contact information, additional navigation links and the authors name.

Commonly used tags & semantic tags used to give meaning to content

Tag Name of Element Description
<h1> to <h6> Heading element Level 1 to Level 6 heading tags convey how important a heading is. A level 1 heading is considered the most important and the level 6 heading is considered the least important.
<p> The Paragraph element This is where blocks of text will usually be placed.
<ol> Ordered List Ordered lists are lists displayed in numerical order. <li> tags are nested within to create the list items.
<ul> Unordered Lists Unordered lists are displayed as bullet point lists in no particular order. <li> tags are nested within to create the list items.
<a> The Anchor element The anchor element is what makes the world wide web interconnected, anthing placed between anchor tags is converted to a link. This link can be directed to any section or page within a website, the link can even be pointed towards any location on the web that has a valid address.
<img> The image element This element is how you insert images into your html document. The src attribute is required to direct the element towards the resource or location the image is to be pulled from. The second attribute alt is not required but very important from an accessibility standpoint.
<section> The Section container element Used to group together a section of content like a blog post or an article. It is considered best practice that each section has a heading.
<article> The article element The Article element is used to group content that makes sense on its own independant of the rest of the page. This could be a blog post or an article that supports the theme of the main page.
<aside> The Aside element The Aside element is used to display content that does not directly relate to the theme of the main page. Usually in the form of a side bar that provides information that compliments the rest of the page e.g a glossary, links to related articles or perhaps an 'about the author' section.
